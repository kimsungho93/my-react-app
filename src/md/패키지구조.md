## 🔍 핵심 폴더별 역할 설명

### 1️⃣ `src/app` : **앱의 뇌/골격**

- **App.tsx**

  - 전체 레이아웃, Router, Provider를 조합하는 루트 컴포넌트

- **providers/**

  - `Redux Provider`, `RouterProvider`, `QueryClientProvider` 같은 것들을 한 곳에 모아 관리

- **store/**

  - `configureStore`, `rootReducer` 등 RTK 스토어 설정

- **routes/**

  - `AppRouter.tsx` 에 `react-router-dom` 라우트 정의

👉 여기까지는 “프로젝트 세팅/인프라 담당”이라고 보면 됨.

---

### 2️⃣ `src/features` : **도메인/기능 단위 묶음 (가장 중요)**

Redux Toolkit까지 쓰고 있으니, **각 기능(도메인)별로 모으는 구조**가 진짜 유용합니다.

예: `counter` 기능

```text
src/features/counter/
├── components/
│   └── CounterPanel.tsx   # 이 기능에서만 쓰는 UI
├── hooks.ts               # 이 슬라이스 전용 훅들 (useCounter 등)
├── counterSlice.ts        # RTK slice
└── index.ts               # public API (외부에서 import 할 때 여기만 보게)
```

장점:

- “카운터 기능 관련 코드가 어디 있지?” → **딱 한 폴더**만 보면 됨.
- 파일이 많아져도 **도메인 단위로 머릿속에 정리가 잘 됨**.
- 나중에 MSA처럼 기능을 떼어낼 때도 구조가 깔끔.

---

### 3️⃣ `src/pages` : **라우트 단위 화면**

- 한 URL 당 하나의 페이지 컴포넌트
  (ex: `/`, `/login`, `/dashboard` 등)
- **페이지는 보통 “조립 역할”**만 하고, 실제 로직은 `features` 쪽에서 가져다 씀.

예)

```tsx
// src/pages/HomePage/HomePage.tsx
import { CounterPanel } from "@/features/counter";
import { MainLayout } from "@/components/layout/MainLayout";

export const HomePage = () => (
  <MainLayout>
    <CounterPanel />
  </MainLayout>
);
```

👉 페이지는 “이 화면에서 어떤 기능들을 보여줄까?”만 고민.
비즈니스 로직/상태관리는 `features`가 담당.

---

### 4️⃣ `src/components` : **완전 공통 UI**

- 도메인에 의존하지 않는 **재사용 가능한 UI 덩어리**

  - Button, Modal, Layout, Input 등

- 다른 프로젝트에서도 그대로 쓸 수 있을 정도가 이상적

---

### 5️⃣ `src/hooks` : **전역 공용 훅**

- 특정 도메인에 종속되지 않는 로직

  - `useWindowSize`, `useDebounce`, `useToggle` 등

- 반대로, 어떤 기능/도메인에 강하게 묶여 있으면 → 그 `feature` 폴더 안에 넣는 게 더 좋음

---

### 6️⃣ `src/api` : **서버 통신 관련**

- `axiosInstance.ts`

  - baseURL, 인터셉터, 공통 헤더 설정

- 각 도메인별 API 모듈

  - `authApi.ts`, `userApi.ts`, `productApi.ts` …

나중에 RTK Query 쓰면:

- `features/auth/authApi.ts` 처럼 각 feature 안으로 옮겨가는 패턴도 있음
- 처음에는 `src/api`로 분리해 두고, 커지면 옮겨도 OK

---

### 7️⃣ `src/types`

- 서버 DTO, 공용 인터페이스, Enum 등
- 중복 선언 막고, API 타입과 프론트 타입을 정리하는 공간

---

### 8️⃣ `src/styles`, `src/assets`

- 전역 CSS, Tailwind 설정, theme 변수 등
- 이미지/아이콘/폰트

## 🧩 경로 지옥 방지용: import alias 추천

`../../../components/common/Button` 이런 거 보기 싫잖아요? 😅
그래서 `@` alias를 많이 써요.

### `tsconfig.app.json` 예시

```json
{
  "compilerOptions": {
    "baseUrl": "./src",
    "paths": {
      "@/*": ["*"]
    }
  }
}
```

그럼 이렇게 쓸 수 있음:

```ts
import { MainLayout } from "@/components/layout/MainLayout";
import { CounterPanel } from "@/features/counter";
import { useAppDispatch } from "@/hooks/useReduxHooks";
```

## 🔚 요약 – “가장 일반적이고 좋은 패턴”

1. **app** : 앱 전체 설정 (Provider, Store, Router)
2. **features** : 도메인/기능 단위 폴더 (RTK slice 포함)
3. **pages** : URL 단위 화면 (여러 feature를 조립)
4. **components** : 완전 공용 UI
5. **hooks/api/types/styles/assets** : 공통 레이어
